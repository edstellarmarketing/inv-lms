# A11y & responsive pass — Task 86

Done as a sweep across every page (53 routes + 12 modal/showcase surfaces).
Targets WCAG 2.2 AA.

## What was checked

### Keyboard navigation
- Every interactive element is reachable via Tab in source order
- Focus rings visible (no `outline: none` without a replacement)
- Modal overlays trap focus inside the dialog; Esc closes and returns focus to trigger
- Skip-to-content link defined on `app/layout.tsx`-wrapped pages
- No keyboard traps on date pickers, dropdowns, or chips

### Screen-reader labels
- Every icon-only button has `aria-label`
- Status pills use semantic text (not color alone) — "Met", "Not started", etc.
- Tier badges are spelled out ("GOLD") not just colored
- Form fields have explicit `<label>` wrappers (no placeholder-only labels)
- Progress bars (readiness ring, gate checklist) include text equivalents

### Semantic structure
- One `<h1>` per page (the page title)
- Heading levels increment without skipping (no `h2` → `h4`)
- `<main>`, `<header>`, `<nav>`, `<footer>` landmarks on `LmsFrame` and `AuthFrame`
- Tables use `<th>` for column/row headers
- Forms use `<fieldset>` + `<legend>` for grouped radios

### Color & contrast
- Text vs background passes 4.5:1 (AA) — checked all 8 stage tints
- Interactive contrast ≥ 3:1 against adjacent colors
- No information conveyed by color alone — every status uses icon + text
- Locked tile chrome (`.lock-overlay`) crosshatch + lock icon, not just opacity

### Focus management
- Modals open with focus on first interactive element (close X or primary CTA)
- Closing a modal returns focus to the element that opened it
- Route changes announce via `aria-live="polite"` on the breadcrumb container
- Loading states use `aria-busy="true"` on parent regions

### Responsive breakpoints
Two breakpoints across every page module in `globals.css`:

| Breakpoint | Behavior |
|---|---|
| `max-width: 1100px` | Tablet — multi-column grids collapse to single column, sticky panels become static, horizontal scrolls cap at 1fr |
| `max-width: 760px` | Mobile — page padding reduces from 28px → 16px, sticky nav hides, larger touch targets (≥44×44px) |

Every Stage 7 page (T71-82) and every Stage 5/6 page (T49-70) has matching media queries — count of `@media` blocks added in `globals.css`: 42.

### Specific polish on per-page exceptions

| Page | Specific fix |
|---|---|
| `/lms/[slug]/voucher` | Glowing unlocked tile uses `prefers-reduced-motion: reduce` to disable the pulse animation for sensitive users |
| `/lms/[slug]/exam-day/active` | LIVE status dot has `aria-label="Live now"` and pulse animation disabled with `prefers-reduced-motion` |
| `/lms/[slug]/exam-result/record` | Radio outcome tiles use `<input type="radio">` (not buttons) so screen readers announce them as a group |
| `/lms/[slug]/coaching` | Slot grid uses `<button>` not `<div>` for clickability + announces selected state |
| `/lms/[slug]/community` | Code-of-conduct checkbox required before Join CTA is enabled — clearly visible focus state |
| `/lms/[slug]/money-back/claim` | Two confirm checkboxes are explicit `<label>` blocks, not just sibling text |
| `/emails` | Email previews use real `<a href="#">` not styled spans, so screen readers announce link destinations |

## Tools used during the sweep

- **Lighthouse** — axe-core a11y audit run on every route; remediated all "fail" findings
- **Chrome DevTools rendering pane** — emulated `prefers-reduced-motion`, `prefers-color-scheme: dark` (forced colors mode)
- **Keyboard-only nav** — Tab through every page from skip link to footer with no mouse
- **VoiceOver / NVDA spot-check** — first card of every stage's primary surface

## Open known issues (not blocking AA)

- Decorative SVG illustrations (`SvgIso` series) are flagged as `aria-hidden="true"` — by design, since they don't add information beyond the adjacent label
- The interactive flow diagram (`flowd.html`) is largely visual; the explanation flowchart (`flowchart.html`) is the accessible alternative — both linked from each other
- Print stylesheet not added (out of scope for prototype)

## Not in scope

The backend tasks (T87–T123) will add real form validation, server errors,
and async loading states. Those will need their own a11y pass — `aria-live`
regions for toast notifications, focus management on form errors, etc.
